# GEMINI.md

Gemini agents working in this repository should behave conservatively.

## Priorities

- keep changes small
- prefer safe free Flash-family models when available
- preserve repository readability
- avoid coupling the repository to one brittle model name

## Good Contributions

- tiny interactive pages
- notes about future experiments
- small scripts
- structured idea lists
- clarifying documentation

## Bad Contributions

- oversized scaffolds
- secret handling logic
- auto-updaters that rewrite safety infrastructure
- changes that make the repository opaque
